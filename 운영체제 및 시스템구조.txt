운영체제란
좁은 의미(커널) : 핵심 부분으로 메모리에 상주하는 부분
넓은 의미(주변 시스템 포함) 

목적 : 자원을 효율적으로 관리, 형평성과 효율성 중심

분류 : 
동시작업 가능 여부 
단일 작업 : MS-DOS 한번에 하나의 작업만 처리
다중 작업 : UNIX,MS Window 동시에 두개 이상 작업 처리
사용자의 수
단일 사용자 : MS-DOS,MS Window
다중 사용자 : UNIX
처리 방식
일괄 처리 : 작업 요청의 일정량을 모아서 한꺼번에 처리(관리자우선)
시분할 : 여러 작업을 수행할 때 일정한 시간단위로 분할, 짧은 응답시간
실시간 : 정해진 시간 안에 어떠한 일이 반드시 종료되야함(deadline)

Multiprocessor : 하나의 컴퓨터에 CPU가 여러 개 붙어 있음.(클라우딩컴퓨팅)

예
UNIX : C언어, 높은 이식성, 최소한의 커널, 확장용이, 소스 공개, 개발 용이
DOS : 단일 사용자용
MS Windows : GUI 기반, 풍부한 지원 소프트웨어

시스템 구조
Mode bit : 사용자 프로그램의 잘못된 수행으로부터 보호하는 장치, 특권 명령으로 규정
1: 사용자 모드 : 사용자 프로그램 수행
0 : 모니터 모드 : 운영체제 수행
timer : 정해진 시간이 흐른 뒤 제어권이 넘어가도록 인터럽트 발생, CPU 독점하는 것을 보호

인터럽트(Interrupt) : CPU 제어권을 운영체제에게 넘겨줌, 인터럽트 당한 시점에서 저장 후 CPU의 제어를 인터럽트 처리 루틴에 넘긴다, Interrupt line을 통해서 처리
Trap(소프트웨어 인터럽트)
Exception : 프로그램이 오류를 범한 경우
시스템 콜 : 사용자 프로그램이 운영체제의 서비스를 받기위해 커널 함수 호출, 프로그램이 인터럽트를 거는 경우

*운영체제는 인터럽트에 의해서 구동됨;

동기식 입출력(synchronous I/O) : 요청한 작업이 끝나고 나서 사용자 프로그램으로 넘어가는 것. 
구현 방법 1 : 끝날 때 까지 기다리면 CPU 낭비
구현 방법 2 : 완료될 때까지 다른 프로그램에서 CPU 사용
비동기식 입출력(Asynchronous I/O) : 기다리지 않고 사용자 프로그램으로 넘어감
*두 경우 모두 끝나는 경우는 인터럽트를 이용해 알려줌

DMA(Direct Memory Access) : 인터럽트를 너무 자주 거는 경우를 제어하기 위해 사용, 빠른 입출력 장치를 메모리에 가까운 속도로 처리하기위해. CPU 중재 없이 메모리에 block 단위로 직접 전송, block 단위로 인터럽트를 발생시킴
