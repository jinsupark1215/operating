운영체제 - 반효경

1강
운영체제의 목적 : 자원을 효율적으로 관리, CPU와 메모리, I/O 장치(하드웨어) 효율적 관리(효율성,형평성)

운영체제의 기능 : CPU 스케줄링(CPU 사용권), 메모리 관리, 디스크 스케줄링, 인터럽트와 캐싱

속도 : CPU > 메모리 > 디스크, IO 장치


CPU 스케줄링 : 
FCFS - 선입선출, 온 순서대로 처리
SJF - CPU 사용시간이 가장 짧은 프로세스 먼저 스케줄(기아 현상 발생 가능,형평성문제)
RR(Round Robin) - 각 프로세스는 동일 크기의 CPU 할당시간을 가짐. 





메모리 관리 : 메모리 꽉찬 공간 이후에 스왑영역에 저장.

메모리 꽉 찼을 때 삭제 방법
LRU : 가장 오래전에 참조 페이지 삭제 
LFU : 참조횟수가 가장 적은 페이지 삭제

디스크 스케쥴링 : 
디스크 접근 시간의 구성 - 탐색시간(헤더 움직이는 시간), 회전지연, 전송시간
SSTF : 기아 문제(형평성) 발생 가능
SCAN : 헤드가 한쪽 끝에서 다른쪽 끝으로 이동하면서 길목에 있는 모든 요청 처리, 한쪽 끝에 도달하면 역방향으로 이동하며 길목의 모든요청처리

캐싱 : 저장되어 있는 걸 빠르게 처리
